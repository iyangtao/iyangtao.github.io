方法引用

本文最后更新于：1 年前

方法引用提供了非常有用的语法，可以直接引用已有Java类或对象（实例）的方法或构造器

概述

  • 当要传递给lambda体的操作，已经有实现的方法了，可以使用方法引用
  • 方法引用本质上就是lambda表达式，而lambda表达式作为函数式接口的实例，所以方法引用也是函数式接口的实例
  • 实现接口的抽象方法的参数列表和返回值类型必须与方法引用的方法的参数列表和返回值类型保持一致

格式

  • 对象 : : 实例方法名
  • 类 : : 静态方法名
  • 类 : : 实例方法名
  • 类 : : new

使用案例

  1. 提供一个公共的Person类，便于测试

  2. 对象 : : 实例方法名

    获取实例对象的年龄

  3. 类 : : 静态方法名

    Person类中定义一个静态方法getNominalAge

    传入一个实际年龄即可获取虚岁

  4. 类 : : 实例方法名

    Person类中定义一个方法ageDiff

    获取两个Person实例的年龄差

  5. 类 : : new

    也称为构造器引用

    通过一个String类型的姓名构造一个Person对象

    这里有个坑：如果Person类中还有一个构造函数，假设为通过String类型的id构造Person

    此时就不能再使用lambda表达式及方法引用，因为Person类中有两个参数为String、返回值为Person的方法，无法判断到底是通过name构造还是通过id构造！

最后

方法引用是代码的简洁之道，但初学可能晦涩难懂，需多加熟悉。
